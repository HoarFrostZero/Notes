1) T的右子结点R连同R的右子树rightchild(R)移动到T的位置

1) T的左子结点L连同L的左子树leftchild(R)移动到T的位置

**问题：对使用频率不同的字符给出没有歧义且长度最小的编码**
前缀编码 任意编码不是其他编码的前缀，顺序读取可直接解码（没有歧义）
假设二叉树有$n$个叶，第$i$个叶带权$w_i$路径长度$l_i$，定义带权路径长度$WPL=\sum_{i=1}^nw_il_i$
则构建Huffman树可以使$WPL$最小：
1) 将所有叶视作树放入堆
2) 循环由堆给出根权最小的两棵树作为新树根的两棵子树，新树根权是两棵子树根权之和，将新树放入堆
3) 直到堆中仅剩一颗树，即为Huffman树
上述过程可以顺带计算$WPL$，在合成新树时累加新树根的权即可
证明：
+ $WPL$最小树是完整二叉树
+ $WPL$最小树最深层存在有相同的父结点的两个叶
+ 将权最小的两个叶与最深层有相同父结点的两个叶交换，$WPL$不增加
1) 显然$n=1$时Huffman树的$WPL$最小
2) 假设$n=k$时Huffman树的$WPL$最小，$n=k+1$时取出权最小的两个叶合成树，归结到$n=k$的情况
Huffman编码 将字符看作叶构建Huffman树，向左的边视为0，向右的边视为1，从根到叶的路径给出叶的编码（易见编码方式不唯一）

## 图 Graph

$G=(V,E)$
$V$是顶点集 $E\subset V^{2}$是边集
赋予权重映射$w:E\to \mathbb{Z}$的$G$称为赋权图
无向边$(u,v)$
有向边$\langle u,v\rangle$ $u$是起点，$v$是终点
出边/入边：顶点作为起点/终点的边
出度/入度：顶点的出边/入边数量
度数：与顶点相关的边数量

路径：边的序列，其中第$i$条边的终点是第$i+1$条边的起点
路径始边的起点称为路径的起点，终边的终点称为路径的终点
路径长度是边权之和（权为1时为边数）
以下约定距离为最短路径长度

环：起点与终点相同的路径
简单路径：只有起点和终点可能相同的路径
迹：端点各不相同的路径

$u$和$v$连通：存在从$u$到$v$的路径
连通无向图：任意顶点连通
强连通有向图：任意顶点连通
弱连通有向图：作为无向图连通
完全无向图：任意顶点存在边
完全有向图：任意顶点双向存在边
连通分量：极大连通子图

图的实现
邻接矩阵
有向图 矩阵的$i$行$j$列记录$\langle i,j\rangle$的权
无向图 矩阵的$i$行$j$列和$j$行$i$列都记录$(i,j)$的权
邻接列表
主列表记录所有顶点，每个顶点关联列表记录出边（可以用字典）

如果邻接矩阵$M\in \mathbf{M}_{n\times n}(\mathbb{Z})$的边权均为$1$，$M^{k}$的$i$行$j$列表示从$i$到$j$长度为$k$的路径数
$\sum_{k=0}^{n-1}M^{k}$穷尽图的所有迹，可用于判断连通分量

**问题：非负权图单源距离**
Dijkstra算法
1) 源点$s$的$dis$为$0$，归入已确定$dis$的顶点集合$D$
2) 已确定$dis$第$k-1$小的顶点$v_{k-1}$，寻找$dis$第$k$小的顶点$v_{k}$，相同$dis$取最先找到的
	$s$到$v_{k}$的最短路径上，考虑$v_{k}$的前驱$p$，由边权非负，$dis(p)\leq dis(v_{k})$，又由$v_{k}$是相同距离最先找到的，$dis(p)<dis(v_{k})$，故$p\in D$
	归纳得$s$到$v_{k}$的最短路径顶点都在$D$中，从$D$的顶点遍历确定$v_{k}$，归入$D$
优化
1) 堆排序给出最近顶点，忽略已确定$dis$的顶点
2) 仅在确定$v_{k}$时更新$v_{k}$的邻接顶点（$dis(v)=min\{dis(v),\,dis(v_{k})+w(\langle v_{k},v\rangle)\}$，称为松弛）并加入堆（只加入变化的顶点）
时间复杂度$O(|E|\log|E|)$

`dis = defaultdict(lambda: float('inf'))`

**问题：一般权图结点距离**
Floyd算法
https://oi-wiki.org/graph/shortest-path/
记顶点为$1\cdots n$，$x$到$y$的路径内点在$1\cdots k$中选择
在$1\cdots k$中搜索时，`d[x][k]`和`d[k][y]`不变，数组无需`k`的维度
`for k in range(1, n + 1):`
	`for x in range(1, n + 1):`
		`for y in range(1, n + 1):`
			`d[x][y] = min(d[x][y], d[x][k] + d[k][y])`
时间复杂度$O(|E|^{3})$，空间复杂度$O(|E|^{2})$

	前者因为加入边$(p,q)$对$p$和$q$增加路径，由顶点与边的数量得环唯一
	后者因为删去环的任意边得到生成树，但原先是最小生成树
定理 相同图最小生成树的边权无交并相同
	对最小生成树$S$与$T$，任意边$e\in S\setminus T$，存在唯一环$R\subset T+e$，环$R$上存在唯一边$f\in T\setminus S$，且$S+f$含相同的环$R$，环$R$上存在唯一边$e\in S\setminus T$，即$e$与$f$互相确定
	由引理$w(e)$和$w(f)$都是环$R$上边权最大值，故存在$S\setminus T\to T\setminus S$的等权双射
推论 边权互不相同的图最小生成树唯一

Prim算法
1) 选定根，循环收集到树最近的顶点，直到$|V|$个顶点
	搜索到树最近的顶点，更新邻接顶点$dis$
+ 暴力循环，时间复杂度$O(|V|^{2})$，**理论**适用于稠密图（实际不一定更快）
+ 堆排序，时间复杂度$O(|E|\log|V|)$，适用于稀疏图
连通图上算法的时间复杂度$|V|^{2}>|E|>|V|$
证明：
1) 无边的树一定在某个最小生成树$T$中
2) 若前$k-1$条边在最小生成树$T$中
	+ 第$k$条边$e$在最小生成树$T$中，成立
	+ 第$k$条边$e$不在最小生成树$T$中，则图$T+e$有含$e$的环，环上存在边$f$未收集，由排序选法$w(e)\leq w(f)\Rightarrow w(T+e-f)\leq w(T)$，故$T+e-f$也是最小生成树，继续考虑$T+e-f$即可

Kruskal算法
1) 循环收集权最小且不成环的边，直到$|V|-1$条边
	堆排序，并查集记录连通顶点
时间复杂度$O(|E|\log|E|)$，适用于稀疏图
+ 并查集的时间复杂度不超过堆排序
证明类似Prim算法

`def next(n, s):`
	`pi = [0 for i in range(n)]`
	`j = 0`
	`for i in range(1, n):`
		`while j != 0 and s[i] != s[j]:`
			`j = pi[j - 1]`
		`if s[i] == s[j]:`
			`j += 1`
	`pi[i] = j`
注意`i`是下标，`j`是真前后缀长度（遗传自上一次循环）
